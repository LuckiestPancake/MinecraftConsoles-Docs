At the moment, this project\textquotesingle{}s scope is generally limited outside of adding new content to the game (blocks, mobs, items). We are currently prioritizing stability, quality of life, and platform support over these things.\hypertarget{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md5}{}\doxysection{\texorpdfstring{Parity}{Parity}}\label{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md5}
We are attempting to keep our version of LCE as close to visual and experience parity with the original console experience of LCE as possible. This means that we will not be accepting changes that...
\begin{DoxyItemize}
\item Backport things from Java Edition that did not ever exist in LCE
\item Swap out LCE visuals for Java Edition or Bedrock Edition style visuals
\item Change LCE defaults in favor of different defaults if it changes the experience
\begin{DoxyItemize}
\item For example, increasing mob spawn limits without increasing the area mobs can spawn within, aka increasing mob density past what was the original console experience
\end{DoxyItemize}
\item Redesign UI components different than LCE
\item Break controller support, or otherwise do not support play with a controller
\item Add custom texture packs or DLC that never existed in LCE
\item Add any gameplay content (block, item, mob) that has no existing point of reference in any official LCE build
\end{DoxyItemize}

However, we would accept changes that...
\begin{DoxyItemize}
\item Fix legitimately buggy or inconsistent behavior in LCE that causes unexpected outcomes
\begin{DoxyItemize}
\item For example, mobs clipping outside of walls, clipping through the world, broken mechanics
\end{DoxyItemize}
\item Add features to better support multi-\/platform use of LCE, such as video and control settings
\begin{DoxyItemize}
\item These menus need to respect the visual style of LCE, though.
\end{DoxyItemize}
\item Replace existing UI systems with SWF-\/free rendering techniques that are as visually and functionally identical as possible
\item Improve the quality of assets (such as sounds) while preserving their contents
\begin{DoxyItemize}
\item For example, upgrading the quality of all music in-\/game while preserving any unique cuts /\+ versions, or faithfully remaking those unique cuts /\+ versions with higher quality assets
\end{DoxyItemize}
\item Backport things like modern skin rendering
\item Change the code from using non-\/stitched textures to individually named texture PNGs and stitching at runtime
\item Adding menus to better support custom dedicated servers with their own fixed IPs
\item Add support for things like Steamworks Networking and other P2P networking and auth strategies
\item Improve \doxylink{class_keyboard}{Keyboard} and \doxylink{class_mouse}{Mouse} control support
\item Add minimal, non-\/invasive Quality of Life features that don\textquotesingle{}t otherwise compromise the LCE experience
\begin{DoxyItemize}
\item For example, adjusting certain crafting recipes or change item behaviors like non-\/stackable doors
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md6}{}\doxysection{\texorpdfstring{Current Goals}{Current Goals}}\label{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md6}

\begin{DoxyItemize}
\item Being a robust Desktop version of LCE
\item Having proper controller support across all types, brands on Desktop or Desktop-\/like platforms (Steam Deck)
\item Improving stability as much as possible
\item Fixing as many bugs as possible
\item Enabling Desktop multiplayer options
\begin{DoxyItemize}
\item LAN P2P Multiplayer
\item Splitscreen Multiplayer
\item WAN Servers (IP:\+Port connectivity)
\item Platform-\/based P2P Connectivity
\begin{DoxyItemize}
\item Steam Networking
\item Game\+Date?
\item Maybe more?
\end{DoxyItemize}
\end{DoxyItemize}
\item Refining rendering settings, renderer options, as well as reaching rendering parity with true LCE
\item Having workable multi-\/platform compilation for ARM, Consoles, Linux
\item Being a good base for further expansion and modding of LCE, such as backports and "{}modpacks"{}.
\end{DoxyItemize}\hypertarget{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md7}{}\doxysection{\texorpdfstring{Scope of PRs}{Scope of PRs}}\label{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md7}
All Pull Requests should fully document the changes they include in their file changes. They should also be limited to one general topic and not touch all over the codebase unless its justifiable.

For example, we would not accept a PR that reworks UI, multiplayer code, and furnace ticking even if its a "{}fixup"{} PR as its too difficult to review a ton of code changes that are all irrelevant from each other. However, a PR focused on adding a bunch of commands or fixes several crashes that are otherwise irrelevant to each other would be accepted.

If your PR includes any undocumented changes it will be closed.\hypertarget{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md8}{}\doxysection{\texorpdfstring{Use of AI and LLMs}{Use of AI and LLMs}}\label{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md8}
We currently do not accept any new code into the project that was written largely, entirely, or even noticably by an LLM. All contributions should be made by humans that understand the codebase.\hypertarget{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md9}{}\doxysection{\texorpdfstring{Pull Request Template}{Pull Request Template}}\label{md__c_1_2_users_2_administrator_2_documents_2_git_hub_2_minecraft_consoles_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md9}
We request that all PRs made for this repo use our PR template to the fullest extent possible. Completely wiping it out to write minimal information will likely get your PR closed.
