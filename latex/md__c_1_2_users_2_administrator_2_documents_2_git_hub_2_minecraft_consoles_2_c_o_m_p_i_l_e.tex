\begin{DoxyEnumerate}
\item Clone or download the repository
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Open the repo folder in Visual Studio 2022+.
\item Wait for cmake to configure the project and load all assets (this may take a few minutes on the first run).
\item Right click a folder in the solution explorer and switch to the \textquotesingle{}CMake Targets View\textquotesingle{}
\item Select platform and configuration from the dropdown. EG:\+ {\ttfamily Windows64 -\/ Debug} or {\ttfamily Windows64 -\/ Release}
\item Pick the startup project {\ttfamily Minecraft.\+Client.\+exe} or {\ttfamily Minecraft.\+Server.\+exe} using the debug targets dropdown
\item Build and run the project:\+
\begin{DoxyItemize}
\item {\ttfamily Build \texorpdfstring{$>$}{>} Build Solution} (or {\ttfamily Ctrl+\+Shift+B})
\item Start debugging with {\ttfamily F5}.
\end{DoxyItemize}

Notes:\+
\begin{DoxyItemize}
\item \doxylink{requirements_The}{The} CMake build is Windows-\/only and x64-\/only.
\item Contributors on mac\+OS or Linux need a Windows machine or VM to build the project. Running the game via Wine is separate from having a supported build environment.
\item Post-\/build asset copy is automatic for {\ttfamily Minecraft.\+Client} in CMake (Debug and Release variants).
\item \doxylink{requirements_The}{The} game relies on relative paths (for example {\ttfamily Common\textbackslash{}Media\textbackslash{}...}), so launching from the output directory is required. 
\end{DoxyItemize}
